# vision.md — タスク管理（ADHD向けPWA）

## 1. Purpose
不注意優勢型ADHDの就活中大学生が、就活・大学・生活の全タスクを一括管理し、アプリを開いた瞬間に「今日何をすべきか」を把握できるPWAタスク管理アプリ。

- 対象ユーザー: 自分自身（1人用）
- 解決課題: 複数企業の選考フロー・大学の講義・日常タスクが混在し、何をいつやるべきか混乱する問題
- 動作環境: Android スマホ（Chrome PWA）

## 2. Features

### F-1. 今日のタスク一覧（ホーム画面）
**User Story:** ユーザーとして、アプリを開いた瞬間に今日やるべきタスクだけを見たい。
**Acceptance Criteria:**
- Given: 今日が締切のタスクが3件ある
- When: アプリを開く
- Then: ホーム画面にカード形式で3件のタスクが時刻順に並んでいる。各カードにタスク名・締切時刻・カテゴリ色ラベルが表示される
- Given: 今日のタスクが0件
- When: アプリを開く
- Then: 「今日のタスクはありません」とイラストまたはメッセージが表示される
**Must NOT:** 未来のタスクがホーム画面に表示されてはいけない

### F-2. タスク登録（手動入力）
**User Story:** ユーザーとして、最小限の操作でタスクを登録したい。
**Acceptance Criteria:**
- Given: ホーム画面にいる
- When: 画面下部の「+」ボタンをタップする
- Then: 入力フォームが表示される。必須項目はタスク名と期限日のみ。カテゴリはワンタップ選択（就活/大学/生活）。期限時刻はオプション
- Given: タスク名「A社 ES提出」、期限「5/10 14:00」、カテゴリ「就活」を入力
- When: 保存ボタンをタップする
- Then: タスクが保存され、該当日のタスク一覧に表示される
**Must NOT:** 登録に3タップ以上の画面遷移が必要であってはいけない

### F-3. タスク登録（音声入力）
**User Story:** ユーザーとして、「明日までにA社のES提出」と喋るだけでタスクを登録したい。
**Acceptance Criteria:**
- Given: ホーム画面にいる
- When: マイクボタンをタップして「明日14時までにA社のES提出」と喋る
- Then: Web Speech APIで音声をテキスト化し、Gemini 2.5 Flash APIでタスク名「A社 ES提出」・期限「明日 14:00」・カテゴリ「就活」を自動解析し、即座にタスクとして保存される
- Given: Gemini APIがレート制限に達している
- When: 音声入力を実行する
- Then: 「AI解析が一時的に利用できません。手動で入力してください」とメッセージが表示され、手動入力フォームが開く
**Must NOT:** 解析結果の確認画面を挟んではいけない（即登録）。誤りは後から編集で修正する

### F-4. タスク完了と報酬演出
**User Story:** ユーザーとして、タスクを完了した時に達成感を感じたい。
**Acceptance Criteria:**
- Given: ホーム画面にタスクカードがある
- When: カードのチェックボックスをタップする
- Then: 紙吹雪アニメーションが再生され、カードに打ち消し線が付き、半透明になる。画面上部のプログレスバー（「3/5 完了！」）が更新される
- Given: 今日の全タスクを完了した
- When: 最後のタスクを完了する
- Then: 特別な祝福アニメーション（大きめの紙吹雪 + 「全完了！」メッセージ）が表示される
- Given: 完了タスクがある状態で翌日になる
- When: アプリを開く
- Then: 前日の完了タスクは表示されない（自動消去）
**Must NOT:** 完了タスクが翌日以降もホーム画面に残ってはいけない

### F-5. ストリーク表示
**User Story:** ユーザーとして、連続でタスクを全完了した日数を見て、継続のモチベーションを保ちたい。
**Acceptance Criteria:**
- Given: 過去3日間、毎日全タスクを完了している
- When: ホーム画面を見る
- Then: 「🔥 3日連続全完了！」のようなストリーク表示がプログレスバーの近くに表示される
- Given: 昨日タスクを1つ残して終了した
- When: ホーム画面を見る
- Then: ストリークは0にリセットされている
**Must NOT:** タスクが0件の日がストリークに加算されてはいけない

### F-6. カレンダー/リスト全体表示
**User Story:** ユーザーとして、未来のタスクの全体像をカレンダーまたはリストで確認したい。
**Acceptance Criteria:**
- Given: 5月に5件、6月に3件のタスクがある
- When: カレンダータブに切り替える
- Then: 月カレンダーが表示され、タスクがある日にカテゴリ色のドットが表示される。日付タップでその日のタスク一覧が表示される
- Given: カレンダー表示中
- When: リスト表示切り替えボタンをタップする
- Then: 全タスクが締切順のリストで表示される。カテゴリでフィルタ可能
**Must NOT:** カレンダーとリストが同時に表示されてはいけない（切り替え式）

### F-7. 締切通知（前日+当日）
**User Story:** ユーザーとして、締切の前日と当日に通知を受け取り、タスクを忘れないようにしたい。
**Acceptance Criteria:**
- Given: 5/10 14:00 締切のタスクがある
- When: 5/9 の朝9:00になる
- Then: 「明日: A社 ES提出（14:00）」という通知が表示される
- Given: 5/10 の朝9:00になる
- When: 当日朝を迎える
- Then: 「今日: A社 ES提出（14:00）」という通知が表示される
- Given: ブラウザが閉じている
- When: 通知時刻になる
- Then: Service Worker経由でローカル通知を試みるが、ブラウザ未起動時は通知が届かない可能性がある（制約として許容）
**Must NOT:** 通知許可ダイアログをアプリ初回起動時に即表示してはいけない（価値を体感した後に許可を求める）

### F-8. 繰り返しタスク
**User Story:** ユーザーとして、毎週・毎月の定期タスクを一度の設定で自動登録したい。
**Acceptance Criteria:**
- Given: タスク登録フォームで「繰り返し」を選択
- When: 「毎週月曜」を選んで保存する
- Then: 毎週月曜にそのタスクが自動で今日のタスクに表示される
- Given: 繰り返しタスクの1回分を完了した
- When: チェックを入れる
- Then: その日のタスクのみ完了扱いになり、次の繰り返し日にはまた未完了で表示される
**Must NOT:** 繰り返しの1回を完了しても、シリーズ全体が完了になってはいけない

### F-9. タスク編集・削除
**User Story:** ユーザーとして、登録済みタスクの内容を修正したり削除したりしたい。
**Acceptance Criteria:**
- Given: タスクカードがある
- When: カードをタップする
- Then: タスク詳細画面が開き、全フィールド（名前・期限・カテゴリ・繰り返し）を編集できる
- Given: タスク詳細画面で削除ボタンをタップする
- When: 確認ダイアログで「削除」をタップする
- Then: タスクが完全に削除される
**Must NOT:** 削除に確認ダイアログなしで即実行されてはいけない

## 3. Pages & UI

### 画面一覧
1. **ホーム画面（/）** — 今日のタスク一覧
2. **カレンダー/リスト画面（/all）** — 全タスク表示（カレンダー↔リスト切替）
3. **タスク登録モーダル** — ホーム画面・カレンダー画面のFABから呼び出し
4. **タスク詳細/編集モーダル** — タスクカードタップで表示

### ホーム画面レイアウト（トップ→ボトム）
- ストリーク表示（🔥 N日連続）
- プログレスバー（3/5 完了！）
- タスクカードリスト（時刻順。各カード: チェックボックス | タスク名 | 時刻 | カテゴリ色ラベル）
- 空状態イラスト（タスク0件時）
- 下部ナビゲーション: ホーム | カレンダー
- FAB（右下）: +ボタン（タスク追加）、マイクアイコン（音声入力）

### デザインシステム
- shadcn/ui ベース
- ライト基調 + 1アクセントカラー（ADHD報酬系に効果的なオレンジ系。達成感・即時報酬との関連が強い）
- カテゴリ色: 就活=ブルー / 大学=グリーン / 生活=パープル
- 完了アニメーション: canvas-confetti ライブラリ

## 4. Data Model（IndexedDB / Dexie.js）

| recurrenceDayOfMonth | number | ❌ | 繰り返し日（monthly時。1-31） |

| フィールド | 型 | 必須 | 説明 |
|---|---|---|---|
| date | string (YYYY-MM-DD) | ✅ | 日付（主キー） |
| allCompleted | boolean | ✅ | その日の全タスク完了フラグ |

## 5. Task Breakdown

### Task 1: プロジェクト初期設定 + PWA設定
- 成果物: next-pwaの設定、manifest.json、Service Worker登録、アイコン設置
- 検証: ブラウザで http://localhost:3000 を開き、「ホーム画面に追加」が可能であることを確認

### Task 2: IndexedDBセットアップ（Dexie.js）
- 成果物: Dexie.jsによるDB定義（Task / Streak テーブル）、CRUD関数
- 検証: Playwright MCPでタスクの作成・読取・更新・削除が動作することを確認

### Task 3: ホーム画面（今日のタスク一覧）
- 成果物: 今日のタスクをカード形式で時刻順表示。プログレスバー。空状態表示
- 検証: ブラウザで http://localhost:3000 を開き、今日のタスクがカード形式で表示されること、タスクが0件の場合は空状態メッセージが表示されることを確認

### Task 4: タスク登録（手動入力）
- 成果物: FABタップでモーダル表示。タスク名・期限日・期限時刻・カテゴリの入力フォーム
- 検証: ブラウザで http://localhost:3000 の「+」ボタンをタップし、タスクを登録して一覧に表示されることを確認

### Task 5: タスク完了と報酬演出
- 成果物: チェックボックスタップで紙吹雪アニメーション、打ち消し線表示、プログレスバー更新。全完了時の特別演出。翌日自動消去ロジック
- 検証: ブラウザでタスクを完了し、紙吹雪アニメーションが再生されること、プログレスバーが更新されること、全完了時に特別演出が表示されることを確認

### Task 6: ストリーク表示
- 成果物: 全タスク完了日のStreakテーブル記録。連続日数計算。ホーム画面への表示
- 検証: ブラウザでストリーク表示が「🔥 N日連続全完了！」の形式で表示されることを確認

### Task 7: タスク編集・削除
- 成果物: カードタップで詳細/編集モーダル。全フィールド編集可能。削除ボタン + 確認ダイアログ
- 検証: ブラウザでタスクカードをタップし、編集・削除が動作することを確認

### Task 8: カレンダー/リスト全体表示
- 成果物: /all ページ。月カレンダー（タスク日にドット表示）。リスト表示切替。カテゴリフィルタ
- 検証: ブラウザで http://localhost:3000/all を開き、カレンダー表示にタスクのドットが表示されること、リスト表示に切り替えられること、カテゴリフィルタが動作することを確認

### Task 9: 音声入力 + Gemini API連携
- 事前調査: Gemini 2.5 Flash APIの最新バージョン・無料枠制限を Brave Search MCPで確認
- 成果物: Web Speech APIで音声→テキスト変換。Gemini 2.5 Flash APIでテキスト→タスク構造化。即登録。レート制限時のフォールバック（手動入力へ遷移）
- 検証: ブラウザでマイクボタンをタップし、「明日14時にA社のES提出」と喋り、タスクが自動登録されることを確認

### Task 10: 繰り返しタスク
- 成果物: 登録フォームに繰り返し設定（毎日/毎週/毎月）追加。繰り返しタスクの自動生成ロジック
- 検証: ブラウザで「毎週月曜」の繰り返しタスクを登録し、該当日にタスクが表示されること、1回分の完了が次回に影響しないことを確認

### Task 11: 締切通知（前日+当日）
- 成果物: Notification API許可フロー（初回は価値体感後に表示）。Service Workerによるスケジュール通知（前日9:00 + 当日9:00）
- 検証: ブラウザで通知許可を与え、テスト通知が表示されることを確認

### Task 12: UI仕上げ + レスポンシブ調整
- 成果物: モバイルファーストのレスポンシブ調整。アニメーション最適化。ダークモード未対応（Out of Scope）
- 検証: ブラウザの開発者ツールでモバイル表示（375px幅）にし、全画面がスマホ画面に収まり、操作しやすいことを確認

### Task 13: Vercelデプロイ確認
- 成果物: Vercelへのデプロイ成功。本番URLでPWAインストール・全機能動作確認
- 検証: デプロイされたURLをAndroidスマホのChromeで開き、「ホーム画面に追加」でインストールし、タスク登録・完了・音声入力・通知が動作することを確認

## 6. Out of Scope
- ダークモード
- マルチデバイス同期
- ユーザー認証/アカウント管理
- データのJSON エクスポート/インポート
- タスクの優先度設定（高/中/低）
- サブタスク機能
- タグ・ラベル機能（カテゴリのみで管理）
- 他ユーザーとのタスク共有
- ランディングページ/OGP

### バックログ
- データバックアップ（JSON エクスポート/インポート）
- タスクの優先度設定
- ダークモード対応
- カスタムカテゴリ追加

## 7. Assumptions
- [ASSUMPTION] [影響度: LOW] アクセントカラーはオレンジ系を採用（ADHD報酬系の即時報酬感と相性が良い色として選定）。合わなければ後から変更可能
- [ASSUMPTION] [影響度: MEDIUM] PWAのローカル通知はService Workerの setTimeout/setInterval ベースで実装する。ブラウザが完全に閉じている状態では通知が届かない可能性がある（サーバーレス構成のため、サーバーサイドプッシュ通知は使用不可）
- [ASSUMPTION] [影響度: LOW] ストリークのリセットは「タスクが1件以上ある日に、未完了が1件以上残った場合」とする
- [ASSUMPTION] [影響度: LOW] 通知の時刻は前日・当日ともに朝9:00固定とする

### 制約: Gemini API無料枠（2026年4月時点）
- 2026年4月以降、無料枠はFlashモデルのみ（Proは有料必須）
- Gemini 2.5 Flash: 10 RPM / 250 RPD / 250,000 TPM
- 個人利用の音声タスク登録（1日数十回）には十分
- レート制限到達時は手動入力にフォールバックする設計
- APIキーは .env.local に保存（NEXT_PUBLIC_GEMINI_API_KEY）
